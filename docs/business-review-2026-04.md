## 1. 시장 환경 분석

### 1.1 한국 출생률 반등 — 9년 만의 전환점

| 연도 | 출생아 수 | 합계출산율(TFR) | YoY |
|------|----------|----------------|-----|
| 2023 | 235,039 | 0.72 | - |
| 2024 | 242,334 | 0.75 | +3.1% |
| 2025 | 254,457 | 0.80 | +6.7% |

- 2024년: **2015년 이후 9년 만에 첫 출생아 수 증가**
- 2025년: 2021년 이후 최다 신생아 수 기록
- 2025년 Q1: +7.3% — 1981년 통계 시작 이래 최대 Q1 증가율
- 2024년 혼인 건수: 222,422건 (+14.9% YoY) — 1970년 이후 최대 증가
- 평균 출산 연령: 33.7세 (밀레니얼 세대)

**시사점**: 출생률 반등으로 타겟 시장이 확대되고 있으며, 33.7세 평균 출산 연령은 모바일-네이티브 밀레니얼 세대가 핵심 고객임을 의미

## 4. Destiny Baby 서비스 특징 및 SWOT 분석

### 4.1 서비스 핵심 특징

Destiny Baby는 **"AI 사주 기반 작명 + 게이미피케이션 + 일일 콘텐츠"**를 결합한 유일한 서비스로, 경쟁사 대비 다음과 같은 차별점을 보유:

| 특징 | Destiny Baby | 넴유베 | 점신 | 마미톡 |
|------|-------------|--------|------|--------|
| AI 작명 | O (LLM 기반) | X (규칙 기반) | X | X |
| 사주 분석 | O (4주, 십성, 호환성) | 부분적 | O | X |
| 카드 가챠 | O (B/A/S/SS/SSS, 피티) | X | X | X |
| 일일 콘텐츠 | O (6종 카드) | X | O (운세) | X |
| 커뮤니티 | O | X | X | O |
| 이름 투표 공유 | O | X | X | X |
| TTS 음성 | O (ElevenLabs) | X | X | X |
| PDF 보고서 | O | X | X | X |
| 출석체크/스트릭 | O | X | X | X |

### 4.2 핵심 특장점 (Strengths)

1. **유일한 "AI + 사주 + 게이미피케이션" 결합체**
   - 경쟁사 108개 중 이 세 가지를 모두 결합한 서비스 없음
   - 작명은 원타임, 운세는 반복 — 둘을 연결한 최초의 시도

2. **깊은 도메인 알고리즘**
   - 사주 계산 엔진: 4주(年月日時), 십성, 오행 밸런스, 부모-자녀 호환성
   - 작명 엔진: 8,000+ 한자 획수 DB, 불용 한자 DB, 자원오행, 81수리, KOSIS 인기도 통계
   - 15개 이상의 전문 라이브러리 (200KB+ 데이터)

3. **높은 기술 스택 품질**
   - Next.js 14 + TypeScript strict (0 에러) + tRPC v11 + Supabase
   - 전체 11개 라우터에 Zod 검증 완비
   - 결제(TossPayments), TTS(ElevenLabs), PDF 생성 모두 구현 완료

4. **Phase 2 리텐션 루프 설계**
   - 운명의 조각 (프라그먼트) 화폐 시스템: 무료 획득 + 유료 구매
   - 출석체크 + 스트릭 보너스: D7/D14/D30 리텐션 메커닉
   - 일일 운세 6종 카드: 1프라그먼트(~100원)으로 93.5% 마진

5. **바이럴 기능 내장**
   - 이름 투표 공유: 가족/친구에게 후보 이름 투표 요청 → 자연스러운 바이럴
   - 가챠 카드: SNS 공유 → "내 아이 카드" 자랑

### 4.3 약점 (Weaknesses)

1. **치명적 배포 차단** (CRITICAL)
   - Supabase DB 마이그레이션 미적용 → 결제, 투표, 커뮤니티, 일일운세 모두 작동 불가
   - 실질적으로 MVP 수준의 데모 상태

2. **보안 취약점** (CRITICAL)
   - `NEXT_PUBLIC_SKIP_AUTH=true` → 전체 인증 우회 활성화
   - Next.js CVE-2025-29927 미패치 → 모든 보호 라우트 접근 가능
   - 어드민 역할 검증 없음 → 누구나 관리자 기능 접근

3. **작명 결과 미표시**
   - AI가 생성한 이름이 실제 UI에 렌더링되지 않음 (하드코딩된 풀 사용)
   - 핵심 가치 제안이 실제로 작동하지 않는 상태

4. **LLM 의존 비용 리스크**
   - Upstage Solar Pro 3 via OpenRouter — API 비용이 스케일에 따라 급증
   - 30개 이름 생성 시 다수의 LLM 호출 필요

5. **콘텐츠 깊이 부족**
   - 커뮤니티: MOCK_POSTS 사용 중
   - 카드 이미지: 실제 디자인 미완
   - KOSIS 데이터: 2023년 정적 데이터 (실시간 아님)

6. **단일 로컬 시장**
   - 한국어 전용, 글로벌 확장 구조 미비
   - 사주/한자는 한국/중국/일본에만 적용 가능

### 4.4 기회 (Opportunities)

1. **출생률 반등** (2025 +6.7%) — TAM 확대
2. **MZ 사주 디지털화** — ChatGPT 사주 검색 월 7.5만건, 디지털 운세 수용도 급증
3. **전통 작명소 디지털 전환** — "스마트폰에 손님 빼앗긴 작명소" (한경)
4. **점신 성공 모델** — 830억 매출 증명, 사주 기반 서비스의 수익화 가능성
5. **인접 시장 확장** — 태명, 개명, 반려동물 이름, 형제 이름 궁합
6. **글로벌 한류** — 한국식 이름에 관심 있는 해외 한인/한류팬

### 4.5 위협 (Threats)

1. **점신/포스텔러의 작명 기능 추가**
   - 1,900만/860만 사용자 기반으로 작명 기능 출시 시 즉시 시장 장악 가능
2. **무료 AI 도구의 확산**
   - ChatGPT/Claude가 직접 작명 가능 → 전문 도구의 차별점 약화
3. **저출산 기조 지속 리스크**
   - 반등이 일시적일 수 있음 (정부 정책 효과 → 소진 가능성)
4. **규제 리스크**
   - 가챠 시스템의 도박성 논란 (게임 분류 법적 분석 필요)
5. **LLM API 비용 상승/중단**
   - OpenRouter/Upstage 의존 → 가격 변동 또는 서비스 중단 리스크

## 6. 서비스 개선사항 및 기획 레벨 정리

### 6.1 MUST-HAVE (P0) — 출시 전 필수

#### 6.1.1 보안 패치
- **작업**: `NEXT_PUBLIC_SKIP_AUTH` 제거, `.env.local` 정리
- **작업**: Next.js 14.2.5 → 14.2.35+ 업그레이드 (CVE-2025-29927)
- **작업**: 어드민 라우트에 역할 기반 접근 제어 (RBAC) 추가
- **작업**: 미들웨어 fail-open → fail-close 전환
- **예상 공수**: 2~3일

#### 6.1.2 Supabase DB 마이그레이션 적용
- **작업**: `supabase db push` 로 10개 마이그레이션 일괄 적용
- **작업**: RLS 정책 활성화 확인
- **영향**: 결제, 투표, 커뮤니티, 일일운세, 출석체크, 프라그먼트 모두 언블록
- **예상 공수**: 0.5일

#### 6.1.3 작명 결과 UI 연결
- **작업**: `getResult` 쿼리 출력을 실제 UI 컴포넌트에 바인딩
- **작업**: 하드코딩된 네임풀 제거
- **요구사항**: AI 생성 30개 이름 → 이름카드 목록 → 상세 분석 → 즐겨찾기 → PDF/TTS
- **예상 공수**: 1~2일

### 6.2 HIGH PRIORITY (P1) — 출시 후 2주 내

#### 6.2.1 에러 로깅 체계 구축
- **현황**: 50+ silent catch 블록
- **작업**: 모든 catch 블록에 `console.error` + Sentry/LogRocket 통합
- **기획**: 에러 발생 시 사용자에게 친화적 에러 메시지 + 재시도 버튼
- **예상 공수**: 1일

#### 6.2.2 ESLint 설정
- **작업**: `next lint` 실행, `.eslintrc.json` 커밋
- **규칙**: `@typescript-eslint/no-explicit-any` 강제
- **예상 공수**: 0.5일

#### 6.2.3 결제 보안 강화
- **작업**: TossPayments 웹훅 서명 검증 추가
- **작업**: 결제 멱등성 체크 (orderId 중복 방지)
- **작업**: `addCredits` TOCTOU 레이스 컨디션 수정 (트랜잭션 사용)
- **예상 공수**: 1~2일

### 6.3 MEDIUM PRIORITY (P2) — 출시 후 1개월 내

#### 6.3.1 커뮤니티 실데이터 전환
- **현황**: MOCK_POSTS 사용 중
- **기획 요구사항**:
  - 게시글 CRUD → Supabase 실제 쿼리
  - 카테고리: 임신, 출산, 육아, 작명, 사주, 자유
  - 정렬: 최신/인기/댓글순
  - 신고/모더레이션 실 동작
- **예상 공수**: 2~3일

#### 6.3.2 KOSIS 인기도 데이터 실시간화
- **현황**: 2023년 정적 데이터
- **기획 요구사항**:
  - KOSIS 공공데이터 API 연동
  - 연도별/월별 인기 이름 실시간 반영
  - 캐시 전략: 일 1회 업데이트
- **예상 공수**: 2~3일

#### 6.3.3 카드 이미지 제작 및 가챠 연동
- **현황**: 가챠 UI 완성, DB 스키마 있으나 실제 카드 이미지 미비
- **기획 요구사항**:
  - 최소 30장 MVP 카드 세트 (5등급별 6장)
  - AI 이미지 생성 (Midjourney/DALL-E) 또는 일러스트레이터 의뢰
  - 가챠 확률 슬라이더 → DB 연동
- **예상 공수**: 5~7일 (디자인 포함)

#### 6.3.4 Rate Limiting 추가
- **현황**: LLM 호출, 결제, 카드 풀 무제한
- **기획 요구사항**:
  - LLM: 분당 5회, 일 20회 제한
  - 카드 풀: 분당 10회 제한
  - 결제: 중복 방지 + 분당 3회 제한
- **예상 공수**: 1일

### 6.4 NICE TO HAVE (P3) — 3개월 내

#### 6.4.1 푸시 알림 시스템
- **기획 요구사항**:
  - FCM (Firebase Cloud Messaging) 통합
  - 알림 종류: 일일 운세 리마인더, 출석체크 리마인더, 커뮤니티 답글
  - 시간대: 오전 8~9시 (출근 시간) 또는 오후 9~10시 (육아 후 여유시간)
  - 개인화: 사주 기반 "오늘의 행운 시간" 알림

#### 6.4.2 소셜 로그인 강화
- **현황**: 카카오, 네이버 OAuth 설정만 존재
- **기획 요구사항**:
  - 카카오 로그인 + 네이버 로그인 실 구현
  - 애플 로그인 (iOS 필수)
  - 게스트→회원 전환 플로우 개선

#### 6.4.3 이름 트렌드 대시보드
- **기획 요구사항**:
  - baby-name.kr 스타일의 연도별/지역별 인기 이름 차트
  - 사용자의 후보 이름이 전국에서 몇 위인지 실시간 확인
  - "이 이름을 가진 유명인" 정보

#### 6.4.4 형제 이름 궁합 서비스
- **현황**: sibling-compatibility.ts 구현됨
- **기획 요구사항**:
  - 첫째 이름 입력 → 둘째 이름 추천 시 "형제 궁합" 점수 표시
  - 이름 조합의 발음 조화, 의미 보완, 오행 밸런스 분석

#### 6.4.5 태명 생성 서비스
- **현황**: taemyeong-generator.ts 존재
- **기획 요구사항**:
  - 임신 초기 사용자를 위한 태명 생성 (작명 전 단계)
  - 태명 → 작명으로 자연스러운 전환 퍼널
  - 태명에 맞는 태교 콘텐츠 연계

#### 6.4.6 글로벌 확장 준비
- **기획 요구사항**:
  - i18n 프레임워크 도입 (next-intl)
  - 영어/일본어 작명 기능 확대
  - 재외동포 타겟 마케팅
